Recommended firmware-push sequence

  1. T-7 days · Notify operations. Confirm the push date with the Berkley grow team. Remind them that any node deliberately powered down or out-of-mesh at push time stays on WP4 and can be migrated later — there's no irreversible window.
  2. T-24 h · Telemetry refresh. Pull live battery / signal / last-comm via portal_api for every assigned device, re-bucket into green / lime / amber / red, and surface any node below the 30%-battery or 50%-signal go/no-go threshold so the on-site team can wake / re-seat / move the repeater before the push.
  3. T-2 h · Mesh prep. Confirm repeater H11 ring is fully online and that the sink + gateway are reachable. The 21 repeaters carry the WP5 image to the nodes; they update first.
  4. T-0 · WP4 → WP5 push from gateway H2100003236. Each online device picks up WP5 within the push window. Anything offline simply stays on WP4 — no damage, no warranty issue.
  5. T+24 h · Verify. Pull device list a second time. Any device still on WP4 that should have updated → check it's online / battery / signal; if it's healthy and stuck, re-issue the push to it. Anything still offline → flag for retire-trade-in or bring it online and re-run.
  6. If rollback is needed: revert gateway H2100003236 to WP4 from the AROYA portal and the entire mesh rolls back. There is no time limit on the rollback.
