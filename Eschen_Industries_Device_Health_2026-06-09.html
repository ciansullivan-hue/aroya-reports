How the WP5 push works: The gateway sends WP5 over-the-air to every node that is online at the scheduled push time with battery ≥ 30 % and signal ≥ 50 %. Nodes that are offline simply stay on WP4 — not harmed, not bricked. Bring offline nodes online before the push to include them. Any that miss it can be recovered later via a WP4 gateway revert + re-run with no time limit. Target WP5 version: 2.2.1.0 (all 39 devices already on WP5 are running this version). WP4 fleet splits across two sub-versions: 1.2.2.0 (29 devices) and 1.2.1.0 (12 devices) — both receive the same WP5 push; no separate action needed per sub-version.
Large Dry has 6 of the 17 offline WP4 nodes. All created June 2021. That room will remain on WP4 unless those nodes come online before the push — WP4 and WP5 nodes cannot relay for each other in the same subnet.
The "41 ready for WP5" and the "41 TEROS-12 trade-ins" are not the same 41 devices — it's a numerical coincidence. The firmware update applies to radio nodes (H3xx serial prefix) that carry WP4 firmware. The TEROS-12 trade-in applies to substrate sensors (ter12 model) plugged into those nodes. These are separate physical hardware on the same assembly; replacing or upgrading the sensor does not affect the node's firmware, and vice versa. If you want to retire a sensor before the firmware push, do so via a WP4 gateway revert so the node stays eligible.
