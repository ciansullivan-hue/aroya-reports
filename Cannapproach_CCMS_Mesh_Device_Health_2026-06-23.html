What changed since yesterday

FR6 / FR7 healedThe split-route problem on FR6 self-resolved.
FR6 today shows 3/3 devices online, all anchored cleanly to H2200001038 via repeater H4210005930. FR7 is also healthy (RSSI ≥6, link 100%). The "two-rooms-closest-to-the-new-gateway-have-the-worst-signal" symptom from yesterday has gone away — the mesh re-converged on its own as predicted, no on-site reset required.
FR2 / FR3 DOWNNew silent-rooms condition: FR2 went silent 06-23 13:42, FR3 silent 06-24 00:28.
FR2 (2 nodes) and FR3 (2 nodes) are both anchored to a gateway id (222789) that wasn't in the previous audit. Their last data packet was hours ago — FR2 ~14 h, FR3 ~3 h — well past the 4 h P0 threshold for FR2.
FR3 was also showing a degraded signal-quality reading (rssi 1/8, link 15%) before it stopped reporting, which is a different pattern from the clean drop FR6 had during the original changeover.
This is the room cluster that needs eyes on it today: confirm gateway 222789 is the gateway you intended to use, that it has power and an internet path, and that the FR2/FR3 nodes can be reset / power-cycled to re-attach.
Joe's Demo silentJoe's Demo went silent at 14:11 on 06-23 — all 4 devices offline.
The Joe's Demo gateway (H2110001266) is still ONLINE in AROYA's gateway table but it is anchoring zero online devices. Most likely the gateway needs to be checked at-site, or one of the room nodes was unplugged. The fix is the same as FR2/FR3: confirm the gateway path, then power-cycle the room nodes.

Carrying forward from the prior post-mortem

The original three hypotheses from yesterday — stale Wirepas routes, child-slot contention near the new gateway, and antenna geometry — still apply to mesh churn during gateway swaps. FR6/FR7 played out exactly as the route-reconvergence story predicted (they resolved without intervention as Wirepas re-costed paths over the day). The same fix-list applies to today's silent rooms: reset the affected nodes first, verify the anchor gateway is healthy, and only escalate to slot contention or antenna geometry if a clean reset doesn't hold.
